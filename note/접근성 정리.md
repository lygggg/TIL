이번에 제가 프로젝트을 진행하면서 접근성에 대해서 공부하고 고민했던 부분들을 공유해보겠습니다.

## 웹 접근성이란 무엇인가?
웹 접근성(web accessibility)은 성별, 장애 유무에 상관없이 모두가 평등하게 불편함 없이 웹사이트를 이용할 수 있게 하는 권리를 말합니다. 즉 우리는 웹사이트에서 누구나 정보와 기능에 접근하기 위해서 웹 접근성에 맞게 웹사이트를 설계해야 합니다.

### 모두가 평등하게 불편함없이 라는게 어떤걸 말하는 걸까요?
간단하게 예를들면 일상생활에서 모두가 사용하는 엘리베이터의 버튼에서도 생각해볼 수 있습니다.
![](https://velog.velcdn.com/images/baayoo90/post/0038df40-26c9-4bf3-a702-71e39a9d89e3/image.png)
시각 장애인들은 버튼에 점자가 표시되어 있지 않으면 엘리베이터를 정상적으로 이용하지 못할 가능성이 큽니다. 하지만 점자를 표시하는 배려 하나만으로도 모두가 평등하게 불편함없이 엘리베이터를 이용할 수 있게 됩니다. 위에서 말하는 웹 접근성도 이와 비슷하다고 생각합니다.

## 웹 접근성은 왜 지켜야 하는가
접근성을 왜 지켜야할까요? 접근성을 지켜야 하는 이유보단 일반적으로 사람들이 접근성을 지키는것을 꺼려하는 이유에 대해서 먼저 말해보겠습니다.

#### 접근성을 지키는 것도 비용이다.
우리의 웹 사이트에 접근성을 생각한다는 것은 따로 접근성에 대한 리소스가 필요하고, 그 코드를 작성해야하고, 또 그것들을 관리해야합니다. 비용이 들어간다는 점에서는 개발자 입장에선 부정적인 사실일수 밖에 없습니다.

그럼에도 우리는 접근성을 지켜야할 이유가 뭐가 있을까요?

#### 비즈니스적인 영향
`세계 보건기구(WHO)에서는 세계 인구 15%이상 (또는 13억명)이 장애를 가지고 있다고 합니다.` 또한 AIR(American Institutes for Research)에 따르면 미국에서 장애가 있는 근로자의 세후 소득은 4,900억 달러라고 합니다. 우리나라도 보건복지부에 따르면 264만 5000명으로 전체 인구 대비 5.1%를 차지한다고 합니다. 우리가 만약 접근성을 무시하게 된다면 앞에서 설명한 고객들의 잠재적 수익을 잃을 수 있습니다.

`일시적인 장애를 가진 사용자가 있을 수 있습니다.` 뼈가 부러져서 일시적으로 손을 사용하지 못한다거나, 언어를 이해하지 못한다거나, 노화로 인해 시력이 떨어져서 웹을 정상적으로 이용할 수 없을 수도 있습니다.

`접근성을 준수한다면 SEO 측면에서 도움이 됩니다.` SEO 봇은 시각 및 청각 같은 일반적으로 사람이 가진 감각을 가지고있지 않으면 오직 키보드로만 탐색하기 떄문입니다. 따라서 웹 사이트에 더 쉽게 엑세스할 수 있다면 더 효과적으로 크롤링됩니다.

#### 법적인 영향
개인적이나, 비즈니스에 영향을 미치는 부분 뿐만아니라 접근성을 준수하지 않았을 경우 생기는 법적인 영향도 생각해야합니다. 현재 많은 나라(캐나다, 영국, 일본, 호주 등등)에서 엄격한 접근성 법률이 존재하며 우리나라 또한 2005년에 웹 접근성 관련 표준이 제정되었습니다.한국정보화진흥원에서는 "어떠한 사용자(장애인, 노인 등), 어떠한 기술 환경에서도 사용자가 전문적인 능력 없이 웹사이트에서 제공하는 모든 정보에 접근할 수 있도록 보장하는 것"이라고 명시하고 있습니다. 

실제로 접근성때문에 발생한 소송사건이 있었습니다.

## 접근성을 위한 노력
위에서 접근성이 무엇인지, 왜 필요한것인지에 대해서 살펴봤습니다. 그럼 우리 개발자들은 접근성을 위해 필요한 리소스가 무엇이고, 어떻게 해야 접근성을 향상시킬 수 있을까요? (이 포스팅에서는 접근성 지침에 대해서는 다루지 않겠습니다. 궁금하시다면 여기를 참고해주세요 [How is digital accessibility measured?](https://web.dev/learn/accessibility/measure)

### ARIA란 무엇인가
ARIA는 진정한 프로그래밍 언어는 아니지만 웹 접근성을 높이기 위해 HTML 요소에 추가할 수 있는 속성 집합입니다.

> Accessible Rich Internet Applications Suite인 WAI_ARIA는 장애가 있는 사용자가 웹 콘텐츠 및 웹 애플리케이션에 보다 쉽게 접근할 수 있도록 하는 방법을 정의한다. 특히 HTML, JavaScript 및 관련 기술로 개발된 동적 컨텐츠 및 고급 사용자 인터페이스 컨트롤을 지원한다.

ARIA는 HTML과 더불어 스크린 리더 같은 AT(보조 기술)와 관련해 디지털 제품에 대한 접근 성을 높이는 데 중요한 역할을 합니다. 둘다 콘텐츠를 점자 또는 텍스트 음성와 같은 대체 형식으로 변환하는데 사용됩니다.

즉 정리하자면 ARIA는 모두가 평등하게 우리의 웹사이트에 접근할 수 있도록 필수적인 요소라고 할 수 있습니다.

## 접근성 트리
그럼 ARIA가 접근성을 향상시키는데에 필수적인 요소가 되는지에 대해서 알아보겠습니다.

![](https://velog.velcdn.com/images/baayoo90/post/65288edb-6f48-4897-abfb-280de9faa689/image.png)

우리는 접근성 tree에 대해서 알아볼 필요가 있습니다. 일단 접근성 tree는 브라우저에서 생성되며, 표준 DOM(문서 객체 모델)트리를 기반으로 합니다. 우리가 실제로 많이들 알고있는 DOM tree와 마찬가지로 접근성 tree는 마크업 요소, 특성 및 텍스트 노드를 나타내는 객체가 포함되어있습니다.

위에서 잠깐 언급했던 스크린리더와 같은 보조기술들은 생성된 접근성 tree를 재귀적으로 순차적으로 탐색합니다.

### 접근성은 왜 사용자의 편리 뿐만 아니라 SEO 측면에도 장점이 있는건가요?





그럼 ARIA 속성을 사용했을때 보조기술(스크린 리더)에 어떻게 읽히게 되는 걸까요? ARIA는 중요한 속성이지만, 요소에 ARIA를 무조건 추가한다고 해서 사용자가 더 쉽게 접근할 수 있는 것은 아닙니다.

> WebAIM Million 접근성 보고서에 따르면 ARIA가 있는 웹사이트는 ARAI가 없는 웹사이트보다 평균적으로 70% 더 많은 오류를 감지했다고 합니다. 이런 오류의 대부분은 ARIA 속성의 부적절한 사용으로 발생했다고 합니다.

### 이미지


#### alt
alt 속성은 이미지의의 대체 텍스트를 명시하기 위한 속성입니다. jpg, png, svg등 파일 형식에 관계없이 alt 속성을 포함시킴으로써 `<img>` 요소에서 간단한 대체 설명을 얻을 수 있습니다. 그리고 스크린리더는 alt 값을 읽어서 사용자에게 이미지가 가지고 있는 정보를 설명합니다.

[카카오페이 기술블로그에서 가져온 코드입니다.](https://tech.kakaopay.com/post/accessibility-stories-for-everyone/)
#### alt 속성을 사용하지 않은 경우
```
  <img src="/img/img_nudge_typeB_320x219.png" />
```
📢 (스크린 리더): 이미지 넛지 타입비 삼백이십엑스이백십구피엔지 이미지

이미지에 alt 속성을 추가하지 않을경우 스크린 리더는 파일이름을 읽습니다. 이것이 우리가 alt 속성을 추가해야하는 이유입니다. 하지만 이미지에도 종류가 있습니다. 정보가 담겨있는 이미지가 있을수 있고, 정보가 필요없는 장식용 이미지일수도 있습니다.

#### 의미가 있는 이미지
```
  <img src="/img/img_nudge_typeB_320x219.png" alt="기뻐하는 죠르디"/>
```
📢 (스크린 리더): 기뻐하는 죠르디 이미지

만약 아무런 정보가 들어가있지 않은 장식용 이미지라면 어떤 정보를 담아야할까요? 일부 이미지에는 담겨있는 정보가 없을수도 있습니다. 이미지 설명이 길거나, 중복이라면 건너뛰도록 할 수 있지만 존재하지 않는 설명은 상상할 수 없습니다. 우리는 의미없는 정보를 제공하면 사용자에게 혼동을 줄 수 있습니다. 그렇기 떄문에 의미가 없는 이미지라면 속성을 비워두는 것이 좋습니다.

#### 장식용 이미지
```
  <img src="/img/img_nudge_typeB_320x219.png" alt=""/>
```
